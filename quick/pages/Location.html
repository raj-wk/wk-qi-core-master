Location

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        address 0..1 Address
      Physical location.
        alias 0..* List<String>
      A list of&nbsp;alternate names that the location is known as, or was known as in the past.
        description 0..1 String
      Description of the Location, which helps in finding or referencing the place.
        endpoint 0..* List<Reference>
      Technical endpoints providing access to services operated for the location.
        identifier 0..* List<Identifier>
      Unique code or number identifying the location to its users.
        managingOrganization 0..1 Reference
      The organization responsible for the provisioning and upkeep of the location.
        mode 0..1 code
      Indicates whether a resource instance represents a specific location or a class of locations.

      Binding: LocationMode (Required)
        name 1..1 String
      Name of the location as used by humans. Does not need to be unique.
        operationalStatus 0..1 Code
      The Operational status covers operation values most relevant to beds (but can also apply to rooms/units/chair/etc such as an isolation unit/dialisys chair). This typically covers concepts such as contamination, housekeeping and other activities&nbsp;like maintenance.

      Binding: value set (Preferred)
        partOf 0..1 Reference
      Another Location which this Location is physically part of.
        physicalType 0..1 Concept
      Physical form of the location, e.g. building, room, vehicle, road.

      Binding: LocationType (Example)
        position 0..1 Position
      The absolute geographic location of the Location, expressed using the WGS84 datum (This is the same co-ordinate system used in KML).
        status 0..1 code
      The status property covers the general availability of the resource, not the current value which may be covered by the operationStatus, or by a schedule/slots if they are configured for the location.

      Binding: LocationStatus (Required)
        telecom 0..* List<ContactPoint>
      The contact details of communication devices available at the location. This can include phone numbers, fax numbers, mobile numbers, email addresses and web sites.
        type 0..1 Concept
      Indicates the type of function performed at the location.
