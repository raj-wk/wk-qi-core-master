CommunicationRequest

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        authoredOn 0..1 DateTime
      For draft requests, indicates the date of initial creation. For requests with other statuses, indicates the date of activation.
        authorizedBy 0..1 Extension
      The practitioner who authorized the communication request, usually the same as the orderer when the orderer is a practitioner.
        basedOn 0..* List<Reference>
      A plan or proposal that is fulfilled in whole or in part by this request.
        category 0..* List<Concept>
      The type of message to be sent such as alert, notification, reminder, instruction, etc.

      Binding: CommunicationCategory (Example)
        context 0..1 Reference
      The encounter or episode of care within which the communication request was created.
        groupIdentifier 0..1 Identifier
      A shared identifier common to all requests that were authorized more or less simultaneously by a single author, representing the identifier of the requisition, prescription or similar form.
        identifier 0..* List<Identifier>
      A unique ID of this request for reference purposes. It must be provided if user wants it returned as part of any output, otherwise it will be autogenerated, if needed, by CDS system. Does not need to be the actual ID of the source system.
        medium 0..* List<Concept>
      A channel that was used for this communication (e.g. email, fax).

      (Example)
        note 0..* List<Annotation>
      Comments made about the request by the requester, sender, recipient, subject or other participants.
        occurrence[x] 0..1 DateTime | interval<DateTime>
      The time when this communication is to occur.
        orderedBy 0..1 Extension
      Agent that ordered the communication.
        payload 0..* List<Payload>
      Text, attachment(s), or resource(s) to be communicated to the recipient.
        priority 0..1 code
      Characterizes how quickly the proposed act must be initiated. Includes concepts such as stat, urgent, routine.

      Binding: RequestPriority (Required)
        reasonCode 0..* List<Concept>
      Describes why the request is being made in coded or textual form.

      (Example)
        reasonReference 0..* List<Reference>
      Indicates another resource whose existence justifies this request.
        reasonRejected 0..1 Extension
      The reason associated with the rejection of this communication request by the receiver. Applicable only if the status is rejected.
        recipient 0..* List<Reference>
      The entity (e.g. person, organization, clinical information system, device, group, or care team) which is the intended target of the communication.
        replaces 0..* List<Reference>
      Completed or terminated request(s) whose function is taken by this new request.
        requester 0..1 Requester
      The individual who initiated the request and has responsibility for its activation.
        sender 0..1 Reference
      The entity (e.g. person, organization, clinical information system, or device) which is to be the source of the communication.
        status 1..1 code
      The status of the proposal or order.

      Binding: RequestStatus (Required)
        subject 0..1 Reference
      The encounter or episode of care within which the communication request was created.
        topic 0..* List<Reference>
      The resources which were related to producing this communication request.
