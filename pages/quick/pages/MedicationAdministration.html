MedicationAdministration

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        category 0..1 Concept
      Indicates the type of medication administration and where the medication is expected to be consumed or administered.

      Binding: MedicationAdministrationCategory (Preferred)
        context 0..1 Reference
      The visit, admission or other contact between patient and health care provider the medication administration was performed as part of.
        definition 0..* List<Reference>
      A protocol, guideline, orderset or other definition that was adhered to in whole or in part by this event.
        device 0..* List<Reference>
      The device used in administering the medication to the patient. For example, a particular infusion pump.
        dosage 0..1 Dosage
      Describes the medication dosage information details e.g. dose, rate, site, route, etc.
        effective[x] 1..1 DateTime | interval<DateTime>
      A specific date/time or interval of time during which the administration took place (or did not take place, when the 'notGiven' attribute is true). For many administrations, such as swallowing a tablet the use of dateTime is more appropriate.
        eventHistory 0..* List<Reference>
      A summary of the events of interest that have occurred, such as when the administration was verified.
        identifier 0..* List<Identifier>
      External identifier - FHIR will generate its own internal identifiers (probably URLs) which do not need to be explicitly managed by the resource. The identifier here is one that would be used by another non-FHIR system - for example an automated medication pump would provide a record each time it operated; an administration while the patient was off the ward might be made with a different system and entered after the event. Particularly important if these records have to be updated.
        medication[x] 1..1 Concept | Reference
      Identifies the medication that was administered. This is either a link to a resource representing the details of the medication or a simple attribute carrying a code that identifies the medication from a known list of medications.

      Binding: value set (Preferred)
        notGiven 0..1 Boolean
      Set this to true if the record is saying that the medication was NOT administered.
        note 0..* List<Annotation>
      Extra information about the medication administration that is not conveyed by the other attributes.
        partOf 0..* List<Reference>
      A larger event of which this particular event is a component or step.
        performer 0..* List<Performer>
      The individual who was responsible for giving the medication to the patient.
        prescription 0..1 Reference
      The original request, instruction or authority to perform the administration.
        reasonCode 0..* List<Concept>
      A code indicating why the medication was given.

      Binding: Reason Medication Given Codes (Example)
        reasonNotGiven 0..* List<Concept>
      A code indicating why the administration was not performed.

      Binding: SNOMED CT Reason Medication Not Given Codes (Example)
        reasonReference 0..* List<Reference>
      Condition or observation that supports why the medication was administered.
        status 1..1 code
      Will generally be set to show that the administration has been completed. For some long running administrations such as infusions it is possible for an administration to be started but not completed or it may be paused while some other process is under way.

      Binding: MedicationAdministrationStatus (Required)
        subject 1..1 Reference
      The person or animal or group receiving the medication.
        supportingInformation 0..* List<Reference>
